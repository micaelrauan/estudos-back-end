# Checklist de Evolução para Desenvolvedor Backend

## Prioridades Essenciais (Diferenciais Relevantes)

### 1. Containerização
- Docker
- Docker Compose
- Criação de ambientes locais reproduzíveis

### 2. Deploy e Publicação de Aplicações
- Deploy em provedores de cloud:
  - AWS
  - Azure
  - Google Cloud
  - Hetzner
- Conceitos básicos de CI/CD

### 3. Testes Automatizados
- Testes End-to-End (E2E)
- Cobertura de testes (coverage)
- Noções de testes de integração

### 4. Banco de Dados Relacional
- SQL avançado
- PostgreSQL
- Uso eficiente de:
  - JOINs
  - Aggregations (GROUP BY, HAVING)
  - CTEs (`WITH`)

### 5. Documentação de APIs
- Swagger / OpenAPI
- API Reference clara e atualizada
- Contratos de API bem definidos

### 6. Tratamento de Erros
- Error handling local (por camada)
- Error handling global (middlewares, filters)
- Padronização de respostas de erro

### 7. Observabilidade
- Logs estruturados
- Monitoramento básico
- Noções de tracing e métricas

---

## Conhecimentos Não Prioritários para a Primeira Vaga

### 1. Arquitetura Avançada
- DDD
- Microserviços
- Arquiteturas complexas

### 2. Bancos de Dados Não Relacionais
- MongoDB
- Redis

### 3. Escalabilidade e Infraestrutura Avançada
- Alta disponibilidade
- Load balancing
- Kubernetes

### 4. Ecossistema de Frameworks
- Evitar aprender muitos frameworks ao mesmo tempo
- Focar em um stack principal antes de explorar alternativas (NestJS, Moleculer, etc.)
